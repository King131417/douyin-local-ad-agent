## 一、准备工作（在 Mac 上执行）

### 1.1 打包项目（精简版，不含 .git 和备份）

```bash
cd ~/Desktop/douyin-local-ad-agent
# 确保 .env 文件已包含最新 auth_code
cat .env | grep OCEAN_ENGINE_AUTH_CODE

# 打包项目（排除 .git 和备份）
tar czf ~/douyin-agent-deploy.tar.gz \
  --exclude='.git' \
  --exclude='data/*.backup*' \
  --exclude='data/*.corrupted*' \
  --exclude='data/*.sql' \
  --exclude='__pycache__' \
  --exclude='*.pyc' \
  --exclude='venv' \
  --exclude='*.log' \
  .
```

打包后大小约 **15-20MB**（不含 .git 和 6 个数据库备份）。

### 1.2 上传到服务器

```bash
# 用 scp 上传到服务器（需要知道 root 密码或已配 SSH 密钥）
scp ~/douyin-agent-deploy.tar.gz root@139.155.140.43:/tmp/

# 同时上传 deploy.sh 脚本
scp ~/Desktop/douyin-local-ad-agent/deploy/deploy.sh root@139.155.140.43:/tmp/
```

> 💡 如果 scp 命令失败，可以用腾讯云控制台的「文件管理」或「OrcaTerm」上传文件。

## 二、服务器端操作（SSH 登录后执行）

### 2.1 SSH 登录服务器

```bash
ssh root@139.155.140.43
# 输入 root 密码（在腾讯云控制台获取）
```

### 2.2 解压项目

```bash
# 创建项目目录
mkdir -p /opt/douyin-local-ad-agent

# 解压
cd /opt/douyin-local-ad-agent
tar xzf /tmp/douyin-agent-deploy.tar.gz

# 检查文件是否完整
ls -la
# 应该看到: ad.py, src/, data/, .env, requirements.txt 等
```

### 2.3 运行部署脚本

```bash
chmod +x /tmp/deploy.sh
bash /tmp/deploy.sh
```

脚本会自动完成：
- ✅ 安装 Python3 + pip
- ✅ 安装项目依赖（flask, requests, apscheduler, pandas）
- ✅ 检查数据库完整性
- ✅ 创建 systemd 守护进程（崩溃自动重启、开机自启）
- ✅ 启动看板服务

### 2.4 配置防火墙（关键！）

在腾讯云控制台操作：
1. 打开 [轻量应用服务器控制台](https://console.cloud.tencent.com/lighthouse/instance)
2. 点击您的实例 → **防火墙** 标签
3. 点击 **添加规则**
4. 填写:
   - 协议: `TCP`
   - 端口: `8888`
   - 策略: `允许`
   - 备注: `Douyin Dashboard`
5. 点击 **确定**

> ⚠️ 如果不配置防火墙，手机/电脑无法访问看板！

## 四、日常维护

### 4.1 查看服务状态

```bash
# 查看服务是否运行
systemctl status douyin-dashboard

# 查看实时日志
journalctl -u douyin-dashboard -f

# 重启服务
systemctl restart douyin-dashboard

# 停止服务
systemctl stop douyin-dashboard
```

### 4.2 Token 续期（每 30 天）

当 `auth_code` 过期时，同步会失败。需要：

1. 在巨量引擎后台获取新的 `auth_code`
2. SSH 登录服务器
3. 编辑 `.env` 文件:
   ```bash
   cd /opt/douyin-local-ad-agent
   nano .env
   # 修改 OCEAN_ENGINE_AUTH_CODE=新的code
   ```
4. 重启服务:
   ```bash
   systemctl restart douyin-dashboard
   ```

### 4.3 查看定时同步日志

```bash
# 定时同步每天 9:30 运行，日志输出到 journal
journalctl -u douyin-dashboard --since "today" | grep "SYNC"
```

### 4.4 手动触发同步

```bash
cd /opt/douyin-local-ad-agent
python3 ad.py 同步
```

## 五、进阶配置（可选）

### 5.1 用 Nginx 反向代理（80 端口访问）

如果您已经有 Nginx 在跑，可以添加一个 location 反代到 8888：

```bash
# 编辑 Nginx 配置
cat >> /etc/nginx/sites-available/default << 'EOF'

server {
    listen 80;
    server_name 139.155.140.43;

    location /douyin/ {
        proxy_pass http://127.0.0.1:8888/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF

nginx -t && systemctl reload nginx
```

然后访问: `http://139.155.140.43/douyin/`

### 5.2 配置 HTTPS（域名 + SSL 证书）

如果您有域名，可以用腾讯云免费 SSL 证书 + Nginx 配 HTTPS：

1. 在腾讯云申请免费 SSL 证书
2. 下载证书到服务器
3. Nginx 配置:
   ```nginx
   server {
       listen 443 ssl;
       server_name your-domain.com;
       ssl_certificate /path/to/cert.pem;
       ssl_certificate_key /path/to/key.pem;
       
       location / {
           proxy_pass http://127.0.0.1:8888;
       }
   }
   ```

## 六、故障排查

| 问题 | 排查方法 |
|------|----------|
| 无法访问 `http://IP:8888` | 检查腾讯云防火墙是否开放 TCP 8888 |
| 服务启动失败 | `journalctl -u douyin-dashboard -n 50` 查看日志 |
| 数据库报错 | 检查 `data/ad_data.db` 是否存在且未损坏 |
| Token 过期 | 查看日志中的 `refresh_token 已失效` 提示 |
| 同步无数据 | 检查 `.env` 中的 `auth_code` 是否有效 |
| 端口冲突 | `lsof -i:8888` 查看谁占用了端口 |
